## Documentation Philosophy

This documentation follows the [Diátaxis framework](https://diataxis.fr/):

- **Tutorials** are learning-oriented lessons that teach by example
- **How-to guides** are goal-oriented recipes for solving specific problems
- **Reference** is information-oriented technical descriptions of the machinery
- **Explanation** is understanding-oriented discussions that clarify and illuminate

Each type serves a different purpose and is optimized for different user needs.

## Contributing to Documentation

Documentation improvements are always welcome! When contributing:

1. **Place documents in the right category** (reference, guides, explanation, tutorials)
2. **Follow the existing style** (active voice, plain English, code examples)
3. **Test code examples** to ensure they work
4. **Update cross-references** if you rename or move files
5. **Submit a pull request** with clear description of changes

See [CONTRIBUTING.md](../CONTRIBUTING.md) for full contribution guidelines.
